# אתר הבחירות של נוער הדמוקרטים 2026

אתר סטטי (React + TypeScript + Vite) למערכת הבחירות לראשות תנועת הנוער.
האתר כולו בעברית, בכיוון RTL, ומעוצב קודם כול למסכי טלפון.

עמוד הבית "חי": הוא מציג ספירה לאחור למועד הקרוב, ועובר **לבד** בין שלבי
הבחירות לפי השעון (בשעון ישראל, `Asia/Jerusalem`) — בלי צורך לרענן את הדף.

## מה צריך כדי להריץ

- **Node.js** בגרסה 20.19 ומעלה (או 22.12 ומעלה).
- מנהל החבילות **npm** (מגיע יחד עם Node).

## פקודות

```bash
npm install      # התקנה חד-פעמית של הספריות
npm run dev      # שרת פיתוח מקומי (לצפייה תוך כדי עריכה)
npm test         # הרצת הבדיקות האוטומטיות
npm run build    # בניית גרסה לפרסום (נוצרת תיקיית dist)
npm run preview  # תצוגה מקדימה של הגרסה הבנויה
npm run lint     # בדיקת איכות קוד
```

לאחר `npm run dev` נכנסים בדפדפן לכתובת שמופיעה במסוף (בדרך כלל
`http://localhost:5173`).

## פרסום לאוויר (GitHub Pages)

הפרסום אוטומטי. בכל דחיפה (push) לענף הראשי, GitHub מריץ בעצמו את הבדיקות,
בונה את האתר ומפרסם אותו. ההגדרות נמצאות בקובץ
[.github/workflows/deploy.yml](.github/workflows/deploy.yml) ואין צורך לגעת בו.

הפעלה חד-פעמית (נעשתה כבר): בהגדרות המאגר ב-GitHub, תחת **Settings → Pages**,
בשדה **Source** בוחרים **GitHub Actions**.

> הכתובות באתר בנויות בשיטת HashRouter (עם `#/`), והנתיבים יחסיים — לכן האתר
> עובד כמו שהוא גם מתת-נתיב של GitHub Pages, בלי הגדרות נוספות.

## איפה עורכים את התוכן?

כל התוכן מרוכז בתיקייה אחת — **`src/content`** — ואין צורך לגעת בקוד:

| מה רוצים לעדכן | הקובץ | השדה |
| --- | --- | --- |
| תאריכי ושעות הבחירות | `src/content/electionConfig.ts` | `registryCloseDate`, `debateDate`, `electionDate` וכו' |
| קישור ההצבעה | `src/content/electionConfig.ts` | `votingUrl` |
| טקסט "מי זכאי להצביע" (כ-30 מילים) | `src/content/electionConfig.ts` | `eligibilityText` |
| טקסט "כיצד תתבצע ההצבעה" (כ-40 מילים) | `src/content/electionConfig.ts` | `votingMethodText` |
| המועמדים והמועמדות | `src/content/candidates.ts` | (הוראות מפורטות בקובץ) |
| ועדת הבחירות | `src/content/committee.ts` | — |
| טקסטים של שלבי עמוד הבית | `src/content/phaseCopy.ts` | — |

### תאריכי הבחירות

בקובץ `src/content/electionConfig.ts`. התאריך בפורמט `שנה-חודש-יום`
(למשל `2026-07-22`) והשעה בפורמט `שעה:דקה` (למשל `22:00`), תמיד בשעון ישראל.
הספירה לאחור בעמוד הבית והמעברים בין השלבים מתעדכנים מהשדות האלה אוטומטית.

### הוספת מועמד/ת

1. שומרים את תמונת המועמד/ת בתיקייה `public/candidates`
   (שם קובץ באנגלית קטנות, למשל `dana-cohen.jpg`).
2. פותחים את `src/content/candidates.ts`, מעתיקים את הדוגמה שבהערה
   וממלאים את הפרטים.
3. שומרים — הכרטיס בעמוד הבית, העמוד האישי והתפריט מתעדכנים לבד.

### החלפת תמונת המועמד לדוגמה

התמונות יושבות בתיקייה `public/candidates`. מחליפים/מוסיפים שם קובץ תמונה,
ומפנים אליו בשדה `image` שבכרטיס המועמד/ת (שם הקובץ בלבד, למשל `dana-cohen.jpg`).

## שלבי עמוד הבית

עמוד הבית מציג תוכן שונה לפי המועד הנוכחי (הכול מחושב מהתאריכים ב-`electionConfig.ts`):

1. **עד סגירת ספר הבוחרים** — ספירה לאחור ל־15.7.2026 22:00.
2. **עד עימות המועמדים** — ספירה לאחור ל־18.7.2026 22:00.
3. **עד פתיחת ההצבעה** — ספירה לאחור ל־22.7.2026 10:00.
4. **ההצבעה פתוחה** (22.7.2026 10:00–22:00) — כותרת יום הבחירות, ספירה לאחור
   לסגירה, וכפתור **להצבעה** (יופיע רק כשמוזן קישור תקין ב-`votingUrl`).
5. **ההצבעה הסתיימה** — מסך סיום. אין ספירה ואין כפתור הצבעה.

בכל השלבים הניווט, עמודי המועמדים ועמוד המידע נשארים זמינים כרגיל.

## סימולציית זמן

כדי לבדוק כל שלב בלי לשנות את שעון המחשב, אפשר להוסיף לכתובת פרמטר `now`
עם רגע בשעון ישראל, למשל:

```
http://localhost:5173/#/?now=2026-07-22T09:59:55
```

הזמן המדומה "רץ" קדימה, כך שאפשר לראות את המעבר האוטומטי בין שלבים בלי לרענן.
זה עובד גם באתר שמתפרסם לאוויר — הזמן המדומה משפיע רק על הדפדפן שמוסיף
את הפרמטר, לא על מה שאחרים רואים.

יש גם לוח קטן בתחתית המסך עם קיצורי דרך לכל גבול (רגע לפני/בדיוק/אחרי) וקישור
"חזרה לזמן אמת" — אבל הוא מוסתר כברירת מחדל. כדי לחשוף אותו, לוחצים 3 פעמים
תוך פחות משתי שניות על הכיתוב "מערכת הבחירות 2026" בתחתית מגירת הניווט (התפריט הנפתח).

## מה עוד צריך למלא לפני הפרסום הסופי

שלושה שדות הושארו בכוונה ריקים/זמניים, וכולם בקובץ `src/content/electionConfig.ts`:

1. **טקסט הזכאות להצביע** (כ-30 מילים) — `eligibilityText`.
2. **טקסט אופן ההצבעה** (כ-40 מילים) — `votingMethodText`.
3. **קישור ההצבעה** — `votingUrl` (כפתור "להצבעה" יופיע לבד כשמוזן קישור `https://` תקין וההצבעה פתוחה).

## מבנה הפרויקט

```
src/
  content/     ← שכבת התוכן (זה מה שעורכים)
  components/  ← רכיבים משותפים (סרגל עליון, תפריט צד, ספירה לאחור, כרטיסים)
  pages/       ← עמודי האתר (בית, מועמד/ת, מידע, ועדה, 404)
  utils/       ← לוגיקת שלבי הבחירות, שעון, ועזרי תוכן
  styles/      ← אסימוני העיצוב והסגנון הגלובלי
  assets/      ← הלוגו הרשמי
```
